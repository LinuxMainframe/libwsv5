Free a scene list array returned by \mbox{\hyperlink{library_8h_a07a41289dbce951ffead9d15b467d7c5}{obsws\+\_\+get\+\_\+scene\+\_\+list()}}.When you call \mbox{\hyperlink{library_8h_a07a41289dbce951ffead9d15b467d7c5}{obsws\+\_\+get\+\_\+scene\+\_\+list()}}, it allocates memory for the scene names and the array itself. You must free all of this using this function when done. Don\textquotesingle{}t try to free the individual strings manually or use plain free() -\/ that won\textquotesingle{}t work correctly.

Why have a special function for this instead of just free()? Because the memory layout needs special handling -\/ there\textquotesingle{}s an array of pointers, each pointing to separately allocated strings.

{\em scenes} & Array of scene name strings (from obsws\+\_\+get\+\_\+scene\+\_\+list) \\
\hline

\end{DoxyParams}
\begin{DoxyNote}{Note}
Safe to call with NULL scenes pointer (does nothing). 

After calling this, the scenes pointer is invalid -\/ don\textquotesingle{}t use it anymore. 

The count parameter must match what \mbox{\hyperlink{library_8h_a07a41289dbce951ffead9d15b467d7c5}{obsws\+\_\+get\+\_\+scene\+\_\+list()}} returned.
\end{DoxyNote}
